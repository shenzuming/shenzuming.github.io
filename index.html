DApp

DApp(Decentralized Application), is an Internet Application, the server runs on a decentralized peer-to-peer network, and the code is open source.

What is Distributed Applications?

Distributed Applications mean that applications are distributed accross multiple servers, not just one. Distributed is necessary when applicaton data and traffic becomes huge and application downtime is unbearable. In Distributed applications, data is backed up in multiple servers for high availability.

The advantages of DApp

  1. DApp can be fault tolerant and there is no single point of failure.
  2. Able to prevent interference from a single mechanism. Because there is no centralized mechanism in this DAPP.
  3. Users can easily trust this application.

The disadvantages of DApp

  1. It is difficult to modify bug or version update. Because every point in the Internet needs to be update.
  2. Some applications require authenticated user’s identity.
  3. It is difficult to develop a DApp, because they need to apply a complex protocol to reach a consensus.
  4. Applications are usually independent of third-party API to get or store data.

Access Centralization Application

Due to a single point of failure, DApp cannot depend on centralization application. But in some cases, DApp needs to get data from a centralizaion application, the main problem is whether the data in the centralization application is correct. To solving this problem, Ethereum accepts ‘Oraclize’ server as the middleman. Oraclize provides ‘TLSNotary’ verification for data.

Hash Function

In cryptography, we define the input as any information or file, and the output is a fixed-length function called “Hash Function”. For example, in equation H(x)=Z, ‘x’ represents the information or the file while ‘Z’ means the hash value of ‘x’.

So the general hash function has the following properties:

  1. Input is an arbitrary size string.
  2. Output is a fixed-length string.
  3. The calculation is in a resaonable amount of time.

While, if we needs a secure hash function, we also require another three important properties:

1. Collision-resistance:

Collision-resistance means if we cannot find two different input ‘x’ and ‘y’, that the output H(x)=H(y), we call the hash function has collision-resistance.

2. Hiding:

Hiding means if we only know the hash value ‘y’, it is not easy to find ‘x’ to satisfied ‘y=H(x)’.

3. Puzzle-friendliness:

For any n-bit output value ‘y’, if there is no feasible way to find ‘x’ in a much smaller time than 2^n, to guarantee the ‘y=H(x)’ establishment. We call it is Puzzle-friendliness.
